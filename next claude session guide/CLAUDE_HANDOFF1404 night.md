## ONE-LINE BRIEF
CatchTheBrief (catchthebrief.com) is an automated daily news briefing site
that publishes 5 AI-curated, in-depth briefs every morning for Indian readers.
Focused on quality over quantity. Zero server. Zero monthly cost.
Runs on GitHub Actions + GitHub Pages.

   Deals code stays in repo but is disabled. Revisit only after 500+ daily readers.

- Picking ONE niche (finance) and going deep
- Writing in simple, jargon-free language with storytelling
- Building audience FIRST, then monetizing via Ditto Insurance
- Using word of mouth + social media + college partnerships
CatchTheBrief should follow a similar "audience first, money later" path.

## NEWS ENGINE — NEW ARCHITECTURE (Session 3)

### Step 1: Fetch wide (15-20 articles)
Pull from multiple India tech/startup RSS feeds:
- TechCrunch India:       https://techcrunch.com/tag/india/feed/
- YourStory:              https://yourstory.com/feed
- Inc42:                  https://inc42.com/feed/
- Entrackr:               https://entrackr.com/feed/
- The Ken (free posts):   https://the-ken.com/feed/
- Gadgets360:             https://feeds.feedburner.com/gadgets360-latest
- Medianama:              https://www.medianama.com/feed/
- Analytics India Mag:    https://analyticsindiamag.com/feed/
Fetch ALL articles from past 24 hours. Target: 15-20 raw articles.

### Step 2: AI-powered ranking (1 Gemini call)
Send all 15-20 headlines + descriptions to Gemini in ONE call.
Prompt: "You are an editor for an Indian tech news site. Rank these articles
by: (a) importance to Indian readers, (b) uniqueness — avoid duplicate topics,
(c) 'would a 25-year-old professional in Bangalore care about this?'
Return the top 5 article indices with a 1-line reason for each pick."

### Step 3: Generate rich briefs (5 Gemini calls)
For each of the top 5, generate an enhanced brief (see BRIEF FORMAT below).

### Step 4: Generate article pages with unique meta tags
Each article.html gets its own <title>, <meta description>, and <meta og:*> tags.

### AI CALL BUDGET (new)
- Article ranking:     1 Gemini call
- Article briefs:      5 Gemini calls
- Total:               6 calls/day (down from 11 — freed up by removing deals)
- Headroom:            14 calls/day unused per key

## BRIEF FORMAT — ENHANCED (Session 3)

Old format: Hook paragraph + 3 bullets + "Why it matters"
New format: Much richer, more content, designed to be the full reading experience.

```
TITLE:        Compelling headline (not the RSS title — AI-rewritten for engagement)
CATEGORY:     e.g. "AI & ML" or "Startup Funding" or "Digital India"
READ_TIME:    e.g. "3 min read"
IMAGE_QUERY:  A search query to find a relevant stock/news image (see Image Strategy)

HOOK:         3-4 sentence story-style opening. Set the scene. Make the reader
              care about what happened and why. Friendly, conversational tone.
              Think "explaining to a smart friend over chai" — not formal news.

CONTEXT:      2-3 sentences of background. What led to this? What's the history?
              This is what makes CatchTheBrief better than a headline — we explain
              the "why" behind the news.

KEY_FACTS:    5 bullet points (not 3) — concrete numbers, names, dates, amounts.
              Each bullet should be a standalone useful fact.
              e.g. "Funding amount: ₹250 crore Series B from Sequoia and Accel"

WHAT_NEXT:    2-3 sentences on what to watch for. What happens next? When?
              This gives readers a reason to come back tomorrow.

WHY_INDIA:    1 sentence — why this matters specifically to Indian readers.

SOURCE:       Original source name + link (for credibility)
```

### Image Strategy
The engine should include a relevant image with each brief. Options:
1. Extract og:image from the source RSS/URL (preferred — most relevant)
2. Use a free stock image API (Unsplash API — free, 50 req/hr)
3. Fallback: category-specific default images stored in /images/defaults/
Images MUST have alt text for accessibility and SEO.

### Fallback hierarchy (never publishes blank):
1. Full AI response (Gemini → Groq)
2. Retry with simpler prompt
3. RSS description as hook + generic facts
4. Last-resort meaningful generic content

## DESIGN OVERHAUL — SESSION 3 PLAN

### Design Philosophy
The current design feels like a generic blog template. The new design should feel
like a modern, premium editorial product — think Quartz, Rest of World, or The Ken.
Clean, lots of whitespace, strong typography, mobile-first.

### Design Principles
1. **Mobile-first** — 80%+ of Indian internet users are on mobile. Design for
   phone screens first, then adapt for desktop. Every element must work on a
   360px wide screen.
2. **Content is king** — no visual clutter. No sidebar on mobile. No ad slots
   (for now). The article text should be the hero.
3. **Whitespace is a feature** — generous padding and margins. Let text breathe.
   Don't cram content edge-to-edge.
4. **Strong typography hierarchy** — headlines should command attention.
   Body text should be effortlessly readable at normal phone distance.
5. **Speed** — no heavy images, no JS frameworks, no web fonts beyond 2 families.
   Target: <2 second load time on 4G (common in India).

### Typography
- Headings:    Space Grotesk (Google Fonts) — modern, geometric, tech-friendly
- Body text:   Inter (Google Fonts) — excellent readability at all sizes
- Mono/code:   JetBrains Mono (if needed for any technical content)
- Base size:   18px body on mobile, 20px on desktop
- Line height: 1.6 for body text (generous for readability)
- Max width:   680px for article text (prevents eye fatigue on wide screens)
