## ONE-LINE BRIEF
CatchTheBrief (catchthebrief.com) is an automated "Content-to-Commerce" static site
that publishes AI-curated news summaries + daily deals every morning for Indian
readers. Zero server. Zero monthly cost. Runs on GitHub Actions + GitHub Pages.

---

## TECH STACK
- Language:     Python 3.11
- AI Primary:   Google Gemini 2.5 Flash (free tier, multiple API keys rotated)
- AI Fallback:  Groq — Llama 3.3 70B (free tier, 14,400 req/day — auto-activates when Gemini quota exhausted)
- Frontend:     Pure HTML + CSS (no framework). Fonts: Playfair Display + DM Sans + DM Mono
- Hosting:      GitHub Pages (free)
- Automation:   GitHub Actions cron job (daily at 3:30 AM UTC = 9:00 AM IST)
- Newsletter:   Buttondown (free, up to 100 subscribers — migrate to Brevo/MailerLite at 80 subs)
- Dependencies: requests, beautifulsoup4, google-genai  (NO lxml — causes Windows build error)
- Analytics:    Google Analytics GA4 (G-V6N03CT88P) — wired into all pages
- AdSense:      ca-pub-2453013968799709 — wired into all pages (3 slots each)
- Affiliate:    Amazon Associates India — tag: catchthebrief-21

## BRIEF FORMAT — HYBRID (updated Session 2)
Old: 3 bullet points only
New: Hook paragraph + 3 quick-scan bullets + "Why it matters" line
- hook: 2-3 sentence story explanation, friendly tone
- bullets: 3 concrete facts/stats/names
- why_it_matters: 1 sentence India angle

Fallback hierarchy (never publishes blank content):
1. Full AI response (Gemini → Groq)
2. Retry with simpler prompt
3. RSS description as hook + generic bullets
4. Last-resort meaningful generic content

## SUGGESTIONS FOR SESSION 3 — DISCUSS & PRIORITISE

### 🔴 HIGH PRIORITY (do these soon)

1. **SEO — Meta tags per article**
   Currently all article pages have the same generic meta description.
   Each article should have its own unique meta title + description (use the hook).
   This is critical for Google indexing individual articles.

2. **robots.txt file**
   The site has a sitemap but no robots.txt.
   Add a simple one so Google crawls correctly:
   ```
   User-agent: *
   Allow: /
   Sitemap: https://catchthebrief.com/sitemap.xml
   ```

3. **AdSense — submit for review**
   Once the site has 2-3 weeks of daily content, submit AdSense for review.
   Replace `data-ad-slot="auto"` with real slot IDs after approval.
   Expected approval time: 1-2 weeks after submission.

4. **Deal RSS feeds — verify they're working**
   The 4 new India deal feeds (91mobiles, MySmartPrice, Smartprix, r/IndiaDeals)
   haven't been tested in production yet. Run the engine and check if deals are
   India-relevant or still showing irrelevant content. May need to swap sources.

### 🟡 MEDIUM PRIORITY (next 2-4 weeks)

5. **Category filter on homepage**
   Add clickable filter buttons (All / Tech / Cricket / Deals etc.) on homepage.

### 🟢 FUTURE / NICE TO HAVE

10. **Price tracking for deals**
    Store deal prices in a JSON file. If same deal appears next day at lower price,
    show a "Price dropped!" badge. Adds real value for deal hunters.

11. **Search bar**
    Simple client-side search across article titles using JS.
    No server needed — just filter cards by text match.

12. **Email digest automation**
    Instead of manually sending newsletters, auto-trigger a Brevo/MailerLite
    email after each successful GitHub Actions build using their API.
    Fully automated morning newsletter with zero manual effort.

13. **Click tracking on deals**
    Add UTM parameters to all deal links so Google Analytics shows which deals
    get clicked. Helps you understand what Indian users actually want.
    e.g. ?utm_source=catchthebrief&utm_medium=deal&utm_campaign=daily

14. **Progressive Web App (PWA)**
    Add a manifest.json + service worker so users can "install" CatchTheBrief

    Especially valuable for Indian mobile-first users.

15. **Flask migration**

    personalised feeds, real-time updates.
