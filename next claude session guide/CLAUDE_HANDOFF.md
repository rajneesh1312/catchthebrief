## ONE-LINE BRIEF
CatchTheBrief (catchthebrief.com) is an automated "Content-to-Commerce" static site
that publishes AI-curated tech news summaries + daily deals every morning for Indian
buyers. Zero server. Zero monthly cost. Runs on GitHub Actions + GitHub Pages.

---

## TECH STACK
- Language:     Python 3.11
- AI:           Google Gemini 2.5 Flash (free tier, multiple API keys rotated)
- Frontend:     Pure HTML + CSS (no framework). Fonts: Playfair Display + DM Sans + DM Mono
- Hosting:      GitHub Pages (free)
- Automation:   GitHub Actions cron job (daily at 9 AM IST)
- Dependencies: requests, beautifulsoup4, google-genai  (NO lxml — causes Windows build error)

## WHAT news_engine.py DOES (step by step)
1. Loads API keys from env vars (GEMINI_API_KEY_1, _2, _3)
2. Fetches 10 news articles via Google News RSS (one per category)
3. For each article: scrapes the full article body from the source URL
4. Sends full body + headline to Gemini → gets back JSON with:
   - "bullets": 3 real facts (not headline rewrites), max 25 words each
   - "hook": 1-sentence India-specific angle, max 20 words
5. Fetches deals from r/IndiaDeals and r/deals Reddit JSON APIs
6. Sorts deals by community upvote score, deduplicates, takes top 6
7. Tags any Amazon.in/Amazon.com links with affiliate tag automatically
8. Builds: index.html, deals.html, articles/*.html, sitemap.xml
9. All pages use {{TEMPLATE_TAGS}} replaced via Python .replace()

## KEY DESIGN DECISIONS (and why)

### Multi-page architecture (not single index.html)
Each article gets its own URL (articles/slug.html) so Google can index
individual pages. Old version overwrote one file daily — all SEO value lost.

### AI reads full article body, not just headline
Old version passed only the headline to Gemini — basically asking it to
rewrite a title. Now we scrape the article body (up to 1500 chars) and ask
Gemini to extract surprising/actionable facts. Much higher quality output.

### API key rotation
Free Gemini tier = 20 requests/day per key, 15/min per key.
Engine accepts 3 keys (GEMINI_API_KEY_1/2/3), auto-rotates when daily quota
hits. Distinguishes between per-minute errors (wait 35s) vs daily quota
errors (rotate key). 3 keys = 60 req/day, enough for 10 articles + retries.

### NO lxml dependency
lxml requires Microsoft C++ Build Tools on Windows — removed it entirely.
RSS parsing now uses Python's built-in xml.etree.ElementTree. BeautifulSoup
uses html.parser (also built-in) for article body scraping.

### Windows-compatible timestamp
Python's %-d and %-I format codes crash on Windows. Timestamp is built
manually: day = now.day, hour = now.hour % 12 or 12, etc.

### Affiliate link tagging
Any URL containing "amazon.in" or "amazon.com" gets ?tag=AFFILIATE_TAG
appended automatically. No manual work needed.

### Design aesthetic
Dark background (#0a0a0f), grain texture overlay, ambient glow orbs.
Playfair Display (serif, editorial) for headlines.
DM Sans (clean, modern) for body text.
DM Mono for labels, timestamps, category tags.
Color-coded category accent bars on article cards.
No frameworks — pure CSS variables + custom properties.

## TEMPLATE INJECTION TAGS
These strings in the HTML templates get replaced by the Python engine:

| Tag               | Replaced with                              |
|-------------------|--------------------------------------------|
| {{ALL_ARTICLES}}  | All article card HTML                      |
| {{SIDEBAR_DEALS}} | Top 3 deal cards for homepage sidebar      |
| {{ALL_DEALS}}     | All 6 deal cards for deals.html            |
| {{LAST_UPDATED}}  | Human timestamp e.g. "13 April 2026 at..." |
| {{ISO_DATE}}      | Machine date e.g. "2026-04-13"             |
| {{SITE_URL}}      | https://catchthebrief.com                  |
| {{TITLE}}         | Article title (article.html only)          |
| {{LABEL}}         | Category label (article.html only)         |
| {{COLOR}}         | CSS class e.g. accent-purple               |
| {{HOOK}}          | India angle sentence                       |
| {{SUMMARY}}       | AI bullet HTML (<ul><li>...</li></ul>)     |
| {{SOURCE_NAME}}   | News source name                           |
| {{SOURCE_LINK}}   | Original article URL                       |
| {{PUB_DATE}}      | Publication date from RSS                  |
